##  Project Overview

This project demonstrates the complete backend VLSI design flow for an SPI Slave,
starting from a Verilog RTL description and ending with a fabrication-ready GDSII layout.
It targets the OSU018 open-source standard cell library (0.18µm CMOS process).

### SPI Protocol Summary

SPI (Serial Peripheral Interface) is a synchronous, full-duplex, master-slave
serial communication protocol originally developed by Motorola. It uses 4 signals:

| Signal | Direction     | Description                          |
|--------|---------------|--------------------------------------|
| MOSI   | Master → Slave | Master Out Slave In (data output)   |
| MISO   | Slave → Master | Master In Slave Out (data input)    |
| SCLK   | Master → Slave | Serial clock generated by master    |
| SS/CS  | Master → Slave | Active-low chip select              |

This implementation operates in **SPI Mode 0** (CPOL=0, CPHA=0) — data is
captured on the rising edge of SCLK and shifted on the falling edge.

##  Design Flow (Qflow)
```
Verilog RTL (spi_top.v)
        │
        ▼
  [1] Synthesis      — Yosys → Gate-level netlist (.blif / .v)
        │
        ▼
  [2] Placement      — Graywolf → Standard cell placement (.def)
        │
        ▼
  [3] Pre-Route STA  — Vesta → Setup/hold timing analysis
        │
        ▼
  [4] Routing        — Qrouter → Metal wire routing (.def)
        │
        ▼
  [5] Post-Route STA — Vesta → Timing after routing parasitics
        │
        ▼
  [6] Migration      — RC extraction (extTOspice)
        │
        ▼
  [7] DRC            — Magic → Design Rule Check (drc = 0 ✅)
        │
        ▼
  [8] LVS            — Netgen → Layout vs Schematic (match ✅)
        │
        ▼
  [9] GDS            — Magic → GDSII export for fabrication
```

## Tool Setup

### Prerequisites
```bash
sudo apt-get install qflow yosys graywolf qrouter magic netgen
```

Add to your `~/.bashrc`:
```bash
export PATH=$PATH:/usr/lib/qflow/bin
export QFLOW_TECH_DIR=/usr/share/qflow/tech
```

### Running the Flow
```bash
mkdir ~/qflow_spi && cd ~/qflow_spi
# Copy spi_top.v into this directory
qflow gui
```

Or run the full flow from the command line:
```bash
chmod +x scripts/run_flow.sh
./scripts/run_flow.sh
```
